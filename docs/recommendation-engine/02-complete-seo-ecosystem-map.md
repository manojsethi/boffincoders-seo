# 02. Complete SEO Ecosystem Map

## Goal

Define the larger SEO product ecosystem so future work becomes a connected operating system, not disconnected tools.

The product should eventually support the complete SEO agency workflow:

1. onboarding.
2. crawling.
3. technical audit.
4. content audit.
5. keyword strategy.
6. opportunity discovery.
7. recommendation planning.
8. execution workflow.
9. reporting.
10. monitoring.
11. client feedback.
12. continuous improvement.

## Core Ecosystem Modules

### 1. Project Onboarding

Purpose:

- understand client/site.
- collect goals.
- define target audience.
- connect integrations.
- configure crawl settings.
- classify website type dynamically.

Must support:

- service business.
- SaaS.
- ecommerce.
- NGO.
- education.
- publisher.
- local business.
- healthcare/YMYL.
- portfolio.
- documentation.
- custom website type.

### 2. Crawl And Site Inventory

Purpose:

- discover URLs.
- collect HTML/markdown.
- extract metadata.
- extract links.
- extract schema.
- classify pages.

Must support:

- Cheerio fast crawl.
- Playwright fallback/rendered extraction.
- sitemap discovery.
- crawl modes.
- page importance.
- page role review.
- crawl diagnostics.

### 3. Technical SEO Audit

Purpose:

- find crawl/indexing/metadata/schema/performance issues.

Must include:

- indexability.
- canonical.
- robots.
- sitemap.
- redirects.
- broken links.
- duplicate metadata.
- headings.
- structured data.
- CWV.
- mobile/page experience.
- security basics.

### 4. Content And Intent Audit

Purpose:

- assess whether pages satisfy search intent and business goals.

Must include:

- keyword-page fit.
- content depth.
- topical coverage.
- direct answer clarity.
- E-E-A-T context.
- trust proof.
- CTA clarity.
- page freshness.
- internal link context.

### 5. Keyword Strategy

Purpose:

- decide which keywords matter and what page should target them.

Must include:

- GSC query import.
- manual keyword entry.
- keyword mapping.
- intent classification.
- funnel stage.
- goal alignment.
- wrong-page detection.
- cannibalization.
- quick wins.
- content gaps.
- external volume/difficulty later.

### 6. Recommendation Engine

Purpose:

- convert audit findings and keyword opportunities into exact actions.

Must include:

- issue recommendations.
- keyword/page recommendations.
- content recommendations.
- technical recommendations.
- priority.
- owner.
- validation method.
- expected impact.

### 7. Content Brief And Writing Workflow

Purpose:

- create briefs and draft content for approved opportunities.

Must include:

- keyword target.
- intent.
- page objective.
- suggested title/H1.
- required sections.
- FAQ ideas.
- internal links.
- schema recommendation.
- CTA guidance.
- external AI writing option.
- analyst review.

### 8. Execution And Task Workflow

Purpose:

- turn recommendations into work.

Must include:

- owner assignment.
- due dates.
- status.
- comments.
- handoff notes.
- implementation evidence.
- verification.

### 9. Reporting

Purpose:

- communicate status and value.

Must include:

- initial audit report.
- weekly progress.
- monthly progress.
- fix verification.
- opportunity backlog.
- client-safe view.
- internal view.
- data freshness.

### 10. Monitoring

Purpose:

- catch regressions and track improvement.

Must include:

- scheduled crawl.
- scheduled audit.
- scheduled sync.
- scheduled reports.
- failed job visibility.
- issue re-open detection.
- traffic/ranking drops.
- integration freshness.

### 11. Client Portal

Purpose:

- let clients see progress and provide feedback.

Must include:

- published reports.
- approved dashboards.
- next actions.
- comments.
- approvals.
- no internal noise.

### 12. Agency Operations

Purpose:

- manage 200-300 clients.

Must include:

- workspace dashboard.
- client health score.
- work queues.
- failed jobs.
- overdue reports.
- owner workload.
- top risks.
- top opportunities.

## External Data Ecosystem

Current/near-term:

- GSC.
- GA4.
- PageSpeed/CWV.
- crawl data.

Future optional:

- keyword volume/difficulty provider.
- rank tracker.
- backlink/referring domain provider.
- SERP competitor provider.
- local citation provider.
- Google Business Profile.

## AI Ecosystem

Local AI:

- classification.
- summaries.
- entity extraction.
- page intent.
- issue rewrite.
- grouping.

External AI:

- content writing.
- high-quality briefs.
- advanced strategy.
- client-ready narrative.

## Product Rule

Every module must connect to at least one of:

- project goal.
- page.
- keyword.
- issue.
- recommendation.
- report.
- monitoring.

If a feature does not connect to the execution system, it is probably a distraction.
