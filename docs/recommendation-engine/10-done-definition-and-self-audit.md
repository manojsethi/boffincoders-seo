# 10. Done Definition And Self-Audit

## Done Definition

Recommendation work is not complete because a table or API exists.

It is complete only when an SEO analyst can use the product to decide, assign, execute, and validate real SEO improvements.

## Required Self-Audit Format

```markdown
# Recommendation Phase X Self-Audit

## Implemented

- ...

## Files Changed

- ...

## Workflow Validated

- ...

## Evidence Quality

- ...

## AI Usage

- ...

## Analyst Controls

- ...

## Client Visibility

- ...

## Remaining Gaps

- ...

## Verdict

Pass / Partial / Fail
```

If verdict is Partial or Fail, do not move to the next phase.

## Phase 1 Done

Issue recommendations are done only when:

- top issue types have useful recommendations.
- every recommendation has evidence.
- every recommendation has validation method.
- analyst can approve/edit/reject.

## Phase 2 Done

Keyword-page fit is done only when:

- GSC queries get verdicts.
- wrong page ranking is detected.
- must-improve uses hard evidence.
- recommendations link keywords and pages.

## Phase 3 Done

Page content analysis is done only when:

- page workspace explains content problems clearly.
- missing sections are useful.
- verdicts are evidence-backed.
- recommendations can be created from page analysis.

## Phase 4 Done

Content briefs/writing are done only when:

- briefs are evidence-backed.
- external AI writing is optional and reviewed.
- YMYL requires human expert review.
- draft content links to recommendation.

## Phase 5 Done

Fix plans/workflows are done only when:

- recommendations become tasks.
- tasks have owners/status/due dates.
- implemented fixes can be verified.
- failed validation reopens work.

## Phase 6 Done

Client strategy reports are done only when:

- reports explain recommendations clearly.
- client/internal visibility is respected.
- progress is measurable.
- next actions are obvious.

## Final Product Questions

Before this system is accepted, answer:

1. Does the tool tell me what to fix first?
2. Does it explain the root cause?
3. Does every recommendation have evidence?
4. Does it connect keywords to pages?
5. Does it tell content teams what to write/change?
6. Does it tell developers what to fix?
7. Does it verify completed work?
8. Does it show improvement over time?
9. Can a client understand the progress?
10. Would an agency team use this every week?

If any answer is no, the recommendation engine is not complete.
